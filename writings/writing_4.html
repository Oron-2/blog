Accessing directories or files with spaces in the name

27/12/21

I ran into a small problem after I started using the terminal a bit. I tried to change my current working directory to another, which, however, contained spaces in its name with no success. So, I had to find a way to navigate to that folder because I knew it was not the only one on my computer having spaces. I searched for a solution on the internet, reached to a friend and ended up with three ways to do that:

  • Using TAB

    • Using the TAB autocompletes the folder's full name by adding a backslash, also called an escape character for each whitespace between words which makes zsh read the space as part of the file name.

    If I have a folder with the name "My folder", the command to navigate to it will look like this:

    cd My\ folder

  • Using double quotes

    • Writing down the path's name being either relative or absolute in double quotes works too. There is no need to add an escape character for the spaces when using double quotes:

    "cd My folder"

It's a little thing, but it was an obstacle for me to continue what I was trying to do.
